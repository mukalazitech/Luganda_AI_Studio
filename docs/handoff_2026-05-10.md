### 1. OpenRouter API Integration (Translation)
**File:** `backend/services/translation/openrouter_service.py` ← NEW  
**File:** `backend/services/translation/service.py` ← UPDATED  
**File:** `backend/core/config.py` ← UPDATED  

The translation pipeline now has a **Pass 4.5** — after semantic search fails, it tries OpenRouter before falling back to NLLB-200. Key details:

- Default model: `google/gemma-2-9b-it:free` (free tier, good quality)
- Returns `confidence: 0.75`, `match_type: "neural_api"` — distinct from NLLB-200's `"neural_local"`
- Silently skipped if `OPENROUTER_API_KEY` is not set in `.env` — app never breaks
- 8 second timeout with graceful fallback to NLLB-200 on failure
- In-memory daily spend cap (`OPENROUTER_DAILY_LIMIT_USD`, default $0.10) — resets on server restart
- System prompt: "Return only the translated text. No explanation, no punctuation changes, no added context."

**To activate:** Add to your `.env` file:
```
OPENROUTER_API_KEY=your_key_here
OPENROUTER_MODEL=google/gemma-2-9b-it:free
```
Get a free key at https://openrouter.ai

### 2. Luganda Text-to-Speech (Meta MMS)
**File:** `backend/services/tts/mms_tts_service.py` ← NEW  
**File:** `backend/api/routes/tts.py` ← NEW  
**File:** `backend/main.py` ← UPDATED (tts router registered)  
**File:** `frontend/translate.html` ← UPDATED (🔊 speaker button)  
**File:** `frontend/teach.html` ← UPDATED (🔊 speaker button on flashcards)  

The app can now speak Luganda out loud using Meta's MMS model — the only freely available model that actually speaks Luganda (not a generic voice approximating it).

- Model: `facebook/mms-tts-lug` — Luganda-specific, ~few hundred MB, CPU only (no VRAM needed)
- Lazy-loaded on first request (~5–15s first time, ~1–2s after)
- API endpoint: `POST /api/v1/tts` with `{"text": "...", "lang": "lug"}` → returns WAV audio
- 🔊 button on translate page plays the translated Luganda text
- 🔊 button on teach page plays each flashcard word
- Button shows spinner while loading, disables during playback to prevent double-fires
- 500 character limit enforced server-side

**First time you click 🔊:** it will seem frozen for ~10 seconds while the model downloads and loads. After that it's fast. This is expected behaviour.

**To install the model dependency:**
```bash
pip install transformers scipy
```
The model downloads automatically on first use from HuggingFace.

### 3. Dataset Export Pipeline
**File:** `scripts/export_dataset.py` ← NEW  
**File:** `scripts/process_feedback.py` ← UPDATED (cleaning pass added)  

User feedback and corrections can now be exported as a clean, HuggingFace-compatible dataset.

**Export format** (`data/training/dataset_export_YYYY-MM-DD.jsonl`):
```json
{
  "source": "hello",
  "target": "Oli otya",
  "direction": "en_to_lg",
  "match_type": "correction",
  "verified": true,
  "submitted_at": "2026-05-10T10:23:00Z"
}
```

**Cleaning rules applied:**
- Drop pairs where source OR target is under 3 characters
- Drop pairs where source == target (noise/accidental submissions)
- Drop pairs with empty or null expected_output
- Deduplicate by `(source, direction)` — verified pairs always beat unverified; most recent wins
- Strip whitespace from all fields

**To run:**
```bash
# Preview without writing
python scripts/export_dataset.py --dry-run

# Export to data/training/dataset_export_2026-05-10.jsonl
python scripts/export_dataset.py
```

## Current Pipeline (Full Picture)

```
Input text
  → Pass 1: Exact match         (confidence: 1.00, match_type: exact)
  → Pass 2: Normalized match    (confidence: 0.98, match_type: normalized)
  → Pass 3: Partial match       (confidence: 0.85, match_type: partial)
  → Pass 4: Semantic search     (confidence: variable ≥0.50, match_type: semantic)
  → Pass 4.5: OpenRouter API    (confidence: 0.75, match_type: neural_api)  ← NEW
  → Pass 5: NLLB-200 local      (confidence: variable, match_type: neural_local)
  → not_found
```
