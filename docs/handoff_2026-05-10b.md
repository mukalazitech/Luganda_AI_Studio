A read-only admin dashboard accessible at `/app/admin.html`. Shows 5 cards:

1. **System Health** — API status, ChromaDB connected, OpenRouter key set, TTS deps installed, ChromaDB disk usage
2. **Collections** — Record counts for vocabulary, sentences, grammar, proverbs + total
3. **Feedback Summary** — Total submissions, last submission timestamp, correct/wrong/needs_review counts
4. **Training Data** — Training pairs count, correction pairs count, last export date
5. **Translation Pipeline** — NLLB model loaded status, OpenRouter last successful call timestamp

**Endpoint:** `GET /api/v1/admin/status`

Key design decisions:
- Each data-gathering section wrapped in `try/except` — one failure does not crash the whole endpoint
- `get_collection()` used (not `get_or_create`) — status endpoint is read-only, never creates collections
- `openrouter_last_call_at` tracked in-memory in `openrouter_service.py` via `_last_call_at` module variable + `get_last_call_at()` accessor
