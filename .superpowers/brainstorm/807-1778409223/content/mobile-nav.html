A — Bottom Tab Bar

4–5 persistent tabs at the bottom. Main features always visible. "More" tab hides Reviews, Admin, API Docs. Native iOS/Android app feel. Thumb-friendly.

Pros

  • Always-visible navigation
  • True mobile-first pattern
  • Matches Creatorly aesthetic
  • One-thumb reachable

Cons

  • Biggest structural change
  • Requires "More" overflow for 7 links
  • No desktop equivalent (need different nav for desktop)

B — Hamburger + Drawer (Enhanced)

Keeps the current slide-in drawer. Upgrades the hamburger to a lime-accented button. All 7 links fit naturally. Same nav on desktop and mobile.

Pros

  • Smallest code change
  • Handles all 7 links gracefully
  • Consistent desktop + mobile
  • Less disruptive to existing pages

Cons

  • Less "mobile-first app" feel
  • Nav is hidden by default
  • Less Creatorly-native

C — Hybrid (Bottom Nav mobile / Sidebar desktop)

Bottom tab bar with 4 core tabs on mobile (≤768px). Full sidebar on desktop (≥769px). Secondary links (Reviews, Admin, Docs) accessible via a "⋯" menu in the top bar.

Pros

  • Best mobile UX (thumb-friendly)
  • Keeps desktop sidebar intact
  • Matches Creatorly mobile patterns
  • 4 clean primary tabs

Cons

  • Most code to write
  • Two nav systems to maintain
  • Secondary links less discoverable on mobile

My recommendation: C — Hybrid. Best Creatorly feel on mobile without breaking the desktop experience you already have.
